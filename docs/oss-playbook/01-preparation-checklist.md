# 开源前准备检查清单

> 每个模块启动开源前，逐项确认以下事项。

## 1. 项目评估

- [ ] 明确开源范围：哪些代码开源，哪些不开源
- [ ] 识别所有内部依赖（私有 Maven 仓库、内部框架）
- [ ] 识别所有外部依赖，确认许可证兼容性
- [ ] 评估代码量和工作量
- [ ] 确定开源后的品牌名称（如 deepcover）
- [ ] 确定开源后的包名（如 io.deepcover.agent）
- [ ] 确定 GitHub 组织/仓库地址

## 2. 信息安全预检

- [ ] 扫描内部域名（如 tsign.cn, esign.cn, timevale.cn 等）
- [ ] 扫描内部 IP 地址
- [ ] 扫描内部邮箱
- [ ] 扫描硬编码密钥/Token
- [ ] 扫描内部 API 路径
- [ ] 扫描内部 Maven 仓库地址
- [ ] 扫描内部 Git 仓库地址
- [ ] 确认 .gitignore 覆盖所有敏感文件

## 3. 依赖审计

- [ ] 列出所有 Maven/Gradle 依赖
- [ ] 确认无内部私有依赖残留
- [ ] 确认所有依赖的开源许可证
- [ ] 确认许可证与 Apache 2.0 兼容
- [ ] 确认无 SNAPSHOT 依赖指向内部仓库

## 4. Git 仓库预检

- [ ] 检查 commit history 中的内部邮箱
- [ ] 检查 commit message 中的内部信息
- [ ] 检查是否有大文件（binary/JAR）混入历史
- [ ] 确认 Git 清理策略（squash vs filter-branch）

## 5. 代码质量预检

- [ ] 确认现有测试覆盖率
- [ ] 确认编译是否通过
- [ ] 确认无 System.out 残留（应替换为日志框架）
- [ ] 确认无 TODO/FIXME 中包含内部信息
- [ ] 确认注释中无内部信息

## 6. 文档预检

- [ ] README.md 内容规划
- [ ] 是否需要多语言文档
- [ ] 配置文件模板（.example 文件）
- [ ] LICENSE 文件选择

## 验收标准

以上所有项目都必须有明确的确认结果（通过/不适用/需要修复）后，才能进入实施阶段。
