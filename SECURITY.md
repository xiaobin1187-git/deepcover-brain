# 安全策略

## 报告安全漏洞

如果你发现了 DeepCover Brain 的安全漏洞，请**不要**通过公开的 GitHub Issue 报告。我们希望保护用户安全，请按照以下流程私下报告：

### 报告流程

1. **发送邮件** - 请将安全漏洞详情发送至项目维护团队的安全邮箱
2. **提供信息** - 报告中请包含以下内容：
   - 漏洞类型（如 SQL 注入、XSS、权限绕过等）
   - 漏洞影响的版本范围
   - 复现步骤的详细描述
   - 概念验证代码（如适用）
   - 漏洞的影响范围和潜在风险
3. **等待回复** - 我们将在收到报告后 48 小时内确认，并在 7 个工作日内提供初步评估

### 我们承诺

- 我们将对报告的漏洞进行认真评估和及时修复
- 我们会在修复完成后公开致谢报告者（除非报告者希望匿名）
- 我们不会对善意报告安全漏洞的研究者采取法律行动

### 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | 支持 |
| < 1.0.0 | 不支持 |

### 安全最佳实践

部署 DeepCover Brain 时，请注意以下安全事项：

- 不要在公开仓库中提交数据库密码、API 密钥等敏感信息
- 生产环境请使用独立的数据库账号，并限制权限
- 确保 Redis 和 RocketMQ 的访问权限已正确配置
- 定期更新依赖版本以修复已知安全漏洞
- 使用 HTTPS 协议进行服务间通信
